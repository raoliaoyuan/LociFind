# LociFind 隐私说明

> LociFind 是本地优先的开源软件（MIT OR Apache-2.0）。本说明描述软件实际的数据行为，随实现同步维护；它不是法律文件。工程侧细则见 [docs/privacy-security.md](./docs/privacy-security.md)。

## 一句话

**你的文件名、路径、文件内容、搜索词、索引数据全部留在你的电脑上；LociFind 没有遥测、没有账户体系、默认不联网。**

## 我们不收集什么

- 不上传任何文件名、路径、文件内容、搜索查询词、索引数据、使用统计。
- 没有 telemetry / 崩溃上报 / 用户账户；开发者收不到你的任何数据。

## 什么时候会联网（唯一场景）

- **模型一键下载**：在设置页主动点击下载时，应用从 `huggingface.co` 拉取本地模型文件（GGUF）；主源连接失败时自动改试公开镜像 `hf-mirror.com`（同一文件、同样仅由你触发）。这是唯一的应用内联网行为；你也可以手动放置模型文件，全程离线使用。
- 快速入门里的外部工具链接（如 Everything 的 voidtools 官网）经你的系统浏览器打开，应用自身不访问。

## 你的电脑上存了什么

| 数据 | 位置 | 说明 |
|---|---|---|
| 搜索索引（含文件元数据、文本摘录、向量） | 数据目录 `index.db` | 只索引你配置的目录 |
| 本地模型 | 数据目录 `models/` | GGUF 文件，本地推理 |
| 运行日志 | 数据目录 `locifind.log*` | 本地排障用 |
| 文件操作审计 | 数据目录 `audit.jsonl` | 记录 open/copy/move 等操作，永不上传 |
| 设置 / 用户同义词 / 搜索历史 | 配置目录 `settings.json` / `user-synonyms.yaml` / `search_history.json` | 搜索历史按敏感数据对待 |

## 如何删除

- **隐私页 →「卸载清理」**：二段确认后一键删除索引、模型、日志、审计、搜索历史、用户同义词（设置文件保留）。
- Windows 卸载器自动执行同一份清理（升级安装不清数据）。
- 操作审计与搜索历史也可在设置页单独清除。

## 团队 daemon 形态（locifindd）的边界

`locifindd` 把检索能力经 MCP 暴露给**部署方自己接入的 LLM 客户端**：检索结果片段（及经授权的原文）会流向该客户端——若部署方接入的是云端 LLM，这部分数据将由该服务处理，属部署方的配置决策。LociFind 提供 collection 级访问控制、token 鉴权、全文读取开关（`allow_full_read`）与检索审计留痕，帮助部署方守住边界；daemon 本身仍不向 LociFind 开发者发送任何数据。
