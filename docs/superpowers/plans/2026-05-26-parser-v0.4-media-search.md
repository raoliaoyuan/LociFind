## Design Decisions（写 plan 前已锁）

1. **拆分范围 = 5 个 parsers + 1 个 common**：`parsers/{file_search,media_search,file_action,refine,clarify,common}.rs`。`lib.rs` 仅留顶层 `parse()` dispatcher + clarify 触发函数（dispatch 前置判断）+ `pub use parsers::*`。
2. **共享 helper 放 `parsers/common.rs`**：`word_present` / `is_word_char` / `parse_time_expression` / `parse_time_fields` / `parse_year` / `parse_date_with_before` / `parse_location_with_language` 等被多个 parser 调用的纯函数。
3. **每个 parser 模块自带 `#[cfg(test)] mod tests_xxx`**：原 lib.rs 的 43 个 test 按归属切到对应 parser 文件。Task 0 完成后总 test 数仍为 43（不增不减）。
4. **Root cause 攻击顺序按杠杆**：Task 1 extensions 多余（推测 30-50 case）→ Task 2 media_type 误判 + 英文 stop words（15-25 case）→ Task 3 title vs quality 边界（5-15 case）。**真实数字以 Gemini Task B 分桶为准**，Task 1 完成后用实测数据 calibrate Task 2/3。
5. **Clarify partial 不在 main plan**：Gemini Task A 独立处理（改 `packages/evals` 比较器，不动 parser）。
6. **MVP-17 fallback 不在 main plan**：Codex 独立 worktree 跑可行性检查，不阻塞 v0.4。
7. **不再扩 fixture / 改 schema**：parser 适应 fixture，不反过来。

## Task 1: MediaSearch 多余 extensions（最大杠杆 root cause）

**Files:**
- Modify: `packages/intent-parser/src/parsers/media_search.rs`

**Problem:** v0.5 evals MediaSearch id=13 "找周华健的朋友" expected 无 extensions，实际 parser 输出 `extensions: [mp3, flac, wav, m4a, ape, ogg, aac, wma, aiff]`；id=14 同。**当 `media_type ∈ {audio, video, image, screenshot}` 时，parser 不应再输出 `extensions` 字段** —— extensions 是 fileSearch 概念，MediaSearch 已经用 media_type 表达了格式集合，再列 extensions 是冗余。

**例外：用户显式说具体格式时保留**。比如 "找 mp3 的周华健"，用户说了 mp3 这个具体扩展名，则保留 `extensions: [mp3]`。判定方式：检查原 input 是否包含 `match_extensions` 命中的具体后缀词（如 "mp3" / "flac" / "mp4"），命中才输出。

**策略：** 在 `parse_media_search` 输出 extensions 字段处加 gate：若 `media_type.is_some()` 且 input 不含 `match_extensions` 命中的具体后缀词 → 不输出 extensions。
