# BETA-13 evals 扩到 1000 条（覆盖驱动 v0.9）实现计划

> **For agentic workers:** 走 superpowers:subagent-driven-development——撰写/校验类 task 派 Opus 子 agent 并行；脚手架/合并/修复类 task 主线做。步骤用 `- [ ]` 跟踪。

**Goal:** 建 v0.9 = 1000 条 parser-only 评测集（v0.5 500 逐字 + 500 条覆盖驱动手标 ground-truth），跑出 baseline，低风险修暴露的 gap、设计性 gap 登记。

**Architecture:** v0.5 不动（byte-equal 锚点）。新增 `fixtures/v0.9/coverage-cases.json`（手标，核心产物）+ `generate-evals-v09` 确定性合并出 `fixtures/v0.9/cases.json`。标注遵「语义轴按意图 / 约定轴查约定表」两轴纪律，独立于 parser 当前行为。

**Tech Stack:** Rust（locifind-evals）+ serde_json + clap；标注产物为 checked-in JSON。

**设计依据：** [docs/superpowers/specs/2026-06-03-beta-13-evals-1000-design.md](../specs/2026-06-03-beta-13-evals-1000-design.md)

**关键不变量（reviewer 守门）：**
- 不改 `packages/evals/fixtures/v0.5/**` → `--fixtures v0.5` 维持 **472/26/2 byte-equal**。
- coverage 每条 `expected_intent` 能反序列化为合法 `SearchIntent`。
- `generate-evals-v09` 纯确定性（重跑产物逐字节一致）。
- parser 修复每个都 v0.5 不回归 + 新增针对性单测。

### Task 1: v0.9 脚手架 — 合并子命令 + schema 合法性门 + 空 coverage 文件

**Files:**
- Create: `packages/evals/fixtures/v0.9/coverage-cases.json`（初始 `[]`）
- Modify: `packages/evals/src/bin/fixtures.rs`（加 `GenerateEvalsV09` 子命令 + 合并逻辑）
- Create: `packages/evals/tests/v09_integrity.rs`（合并确定性 + schema-valid + 计数门）

- [ ] **Step 1**：`fixtures.rs` 加子命令 `generate-evals-v09 {--coverage <path>} {--output <path>}`：读 v0.5/cases.json（逐字保留）+ coverage-cases.json，确定性拼接（v0.5 前、coverage 后），断言总数==1000（coverage 满 500 时）、id 全局唯一、coverage 每条 expected_intent serde→`SearchIntent` 成功，写 v0.9/cases.json。
- [ ] **Step 2**：脚手架期 coverage 为 `[]`，合并产出 500 条（仅 v0.5），放宽计数断言为「== 500 + coverage.len()」便于分批。
- [ ] **Step 3**：`tests/v09_integrity.rs`——① coverage 全部反序列化为合法 SearchIntent；② id 全局唯一；③ 合并确定性（连跑两次 in-memory 逐字节一致）。
- [ ] **Step 4**：`cargo test -p locifind-evals` 通过；`--fixtures v0.5` 仍 472/26/2。

### Task 11: 对抗式标签校验

- [ ] 按域派校验子 agent，默认质疑每条 label：① schema 合法 ② variant 正确 ③ 语义轴符合意图 ④ 约定轴符合约定表。存疑标 `disputed` + 理由。
- [ ] 主线裁决 disputed：改标 / 保留 / 丢弃补新。宁丢弃不污染。
