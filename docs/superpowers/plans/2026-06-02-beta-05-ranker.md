# BETA-05 Ranker Implementation Plan

> Steps use checkbox (`- [ ]`). 每 task 末尾过 fmt + clippy(-D warnings) + test。

**Goal:** 新 crate `packages/ranker`：对 fan-out 合并集做相关性启发式排序（默认）+ 显式 sort 跨源生效；集成进 desktop fan-out 路径（收齐→rank→发）。纯启发式、仅 fan-out 路径。

**Architecture:** 见 [spec](../specs/2026-06-02-beta-05-ranker-design.md) §3。`RankContext` + `rank`，纯函数。

**Tech Stack:** Rust；locifind-result-normalizer（MergedResult）+ locifind-search-backend。

---

## Task 1: `packages/ranker` crate — RankContext + rank

**Files:** `packages/ranker/{Cargo.toml,src/lib.rs}`（新）、根 Cargo.toml members。

- [ ] **Step 1:** 根 members 加 `"packages/ranker"`；Cargo.toml 依赖 result-normalizer + search-backend（path）。
- [ ] **Step 2:** `RankContext { keywords, sort }` + `from_expanded(&ExpandedSearchIntent)`：
  - keywords = base FileSearch.keywords / MediaSearch.{keywords,artist,title,album} + expanded.keyword_groups.all()，小写去重；
  - sort = `intent_sort_order(&expanded.base)`。
- [ ] **Step 3:** `rank(Vec<MergedResult>, &RankContext) -> Vec<MergedResult>`：
  - 显式 sort（非 RelevanceDesc）→ 按对应 metadata 字段排（compare_option 语义，缺失末尾），不写 score；
  - 否则 → 算相关性分（`relevance_score` helper：0.5·name_match + 0.3·match_weight + 0.2·source_boost）写入 result.score，按 score 降序 + tiebreak（modified_time desc → name asc）。
  - helpers：`name_match(name, keywords)` / `match_type_weight(&[MatchType])` / `source_boost(len)`。
- [ ] **Step 4:** 单测（spec §4.1）：相关性（name 命中靠前 / Filename>Content / 多源>单源 / score∈[0,1] / tiebreak / 无 keyword 退化 / 空）；显式排序（ModifiedDesc/SizeDesc/NameAsc + 缺失末尾）；from_expanded 提取。
- [ ] **Step 5:** fmt + clippy + test。
- [ ] **Step 6:** Commit `feat(ranker): BETA-05 多源结果相关性启发式排序`。
