## 1. 目标与范围

### 1.1 目标

攻克 v0 的退化解问题（86 次 fallback 触发中 valid_intent 仅 8.3%，adapter 实质 no-op）。让模型真正学到 patch 字段填法，让 hybrid 架构在 fallback 触发时能补救 parser 失败的 case。

**门槛**（与 [启动 spec §10 验收清单](./2026-05-27-beta-08-lora-design.md) 一致，本会话不放宽）：

- **门槛 1**：v0.5 evals `--with-fallback --hybrid` **pass 净增 ≥ 5**（相对 parser-only baseline 460/500）
- **门槛 2**：**regressed ≤ 2**

满足两条 = adapter v1 ready；任一未达 = v1 not_ready，按 [v0 报告 §7(b)](../../reviews/beta-08-lora-v0.md) 升路径 (b) 改训练目标只用 nonempty case。

### 1.2 范围

- **数据**：v0.5-patch/v0 全 498 sample 不动，prepare 脚本运行时动态 oversample nonempty 8×
- **基座**：`~/models/qwen25_1.5b_draft/`（与 v0 一致）
- **训练**：mlx-lm lora，**加 `--mask-prompt` flag**；其他超参完全维持 v0（iters 1000 / num-layers 16 / batch 4 / lr 1e-4 / seed 42）
- **GGUF 路径**：与 v0 完全一致（mlx fuse → HF safetensors → convert_hf_to_gguf.py → fp16 GGUF → llama-quantize Q4_K_M）
- **评测**：与 v0 完全一致（parser-only baseline + `--with-fallback --hybrid` 跑 v0.5 全 500 case）
- **产物路径**：全 `main-v0` 改为 `main-v1`；v0 产物保留作 baseline 对比，不被覆盖
- **出场报告**：`docs/reviews/beta-08-lora-v1.md`

### 1.3 不在范围

- 调超参（lr / batch / num-layers / step）
- 数据集 v1 入库（v0.5-patch/v0 dataset 不动）
- 改 evals 流程 / hybrid 架构 / parser
- 路径 (b)：只用 nonempty case 训练。仅在 v1 not_ready 时启动
- 多 oversample 倍数 sweep
- early stopping / best-checkpoint 选择
- delete / reset v0 任何产物

## 2. 与 v0 的差异（隔离单一变量）

| 维度 | v0 | v1 | 变化原因 |
|---|---|---|---|
| 训练 loss | prompt + completion 全算 | `--mask-prompt`：仅在 completion token 算 loss | v0 报告 §6 诊断：prompt 部分 token loss 拖低梯度信号，让模型把"复述 prompt"也当作目标 |
| 数据 class balance | 443 empty / 55 nonempty (88% empty) | oversample nonempty 8× → 443 empty / 440 nonempty (~50/50) | v0 报告 §6 诊断：88% empty 让"永远输出 `{}`" 成 majority-class 最优解 |
| 超参 | 1000 step / lr 1e-4 / batch 4 / num-layers 16 | **完全不变** | 隔离 v1 唯一变量是 data balance + loss masking；超参变化推迟到 v2 |
| 数据集入库 | v0.5-patch/v0 (498 record) | **不变** | v0.5-patch/v0 dataset 仍是单一权威版本；oversample 是训练管线变换，不是数据集变换 |
| 产物路径 | `main-v0-*` | `main-v1-*` | v0 产物保留作 baseline 对比 |
| run 脚本 | `run_main.sh` | `run_main_v1.sh`（复制后改路径 + 加 flag） | 不动 v0 脚本，便于事后回放 |

## 4. prepare_main_data.py 改动

### 4.1 新增逻辑

```python
NONEMPTY_OVERSAMPLE = 8  # v1: nonempty 重复 8× 以达 50/50 balance

def is_empty_patch(completion: str) -> bool:
    """v0 数据集 empty patch completion 是 "{}" 字面值"""
    return completion.strip() == "{}"

# 在 minimal 构造后、shuffle 前加：
empty = [r for r in minimal if is_empty_patch(r["completion"])]
nonempty = [r for r in minimal if not is_empty_patch(r["completion"])]
oversampled_nonempty = nonempty * NONEMPTY_OVERSAMPLE
all_records = empty + oversampled_nonempty
rng.shuffle(all_records)
# 后续 train.jsonl / valid.jsonl 写入逻辑用 all_records 替换 minimal
```

### 4.2 输出 assertion 加 print

```
✅ main data prepared (v1 oversample):
   empty=443
   nonempty=55 × 8 = 440
   total=883
   valid_tail=50
   outdir=training/mlx-lora/data/main
```

### 4.3 兼容性

- v0 仍可重跑：把 `NONEMPTY_OVERSAMPLE=1` 即等价 v0 行为（empty + nonempty*1 + shuffle = 原 minimal）
- v0 / v1 共用同一个 prepare 脚本，靠 `NONEMPTY_OVERSAMPLE` 控制；不重复实现 prepare_main_data_v1.py

> **决策**：本会话采用"修改 prepare_main_data.py + 在 README 标注 v0 时是 `NONEMPTY_OVERSAMPLE=1`"。回到 v0 行为可通过改常量 + 重跑实现。

## 7. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| **R1**: oversample 让模型对 nonempty 过敏 → regressed 上升 | 中 | 中（门槛 2 拦） | 门槛 regressed ≤ 2 限制；若超阈值，降到 6× / 4× 再试 |
| **R2**: mask-prompt 后 loss 数值无法直接与 v0 比 | 高 | 低 | loss 曲线只看收敛趋势；以 evals pass 数为唯一外部 truth |
| **R3**: v1 仍 not_ready（mask-prompt + oversample 不解决根本问题） | 中 | 中（vibe，但合法结案） | v0 产物保留；按 v0 报告 §7(b) 升路径 (b) 改训练目标只用 nonempty case |
| **R4**: mlx-lm `--mask-prompt` 实际行为异常 | 低 | 中 | 已验证 mlx-lm 0.29.1 支持该 flag；若 loss 曲线出现 NaN 或不收敛先打印 loss 取样定位 |
| **R5**: oversample 后训练耗时显著上升（83% 数据增量） | 低 | 低 | step 不变，单 step 耗时与 v0 持平；总耗时 ≈ v0 的 ~50 min |
